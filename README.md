# ربات تلگرام قیمت ارز و طلا

ربات تلگرام برای نمایش قیمت لحظه‌ای دلار، یورو، پوند، طلا و چند ارز دیجیتال محبوب
(بیت‌کوین، اتریوم، تتر)، با اسکرپ از tgju.org.

## ویژگی‌های این نسخه

1. **کش فایلی به‌جای اسکرپ در لحظه:** یک ترد پس‌زمینه هر ۳۰ ثانیه (`SCRAPE_INTERVAL`
   در `config.py`) قیمت‌ها را از tgju.org می‌گیرد و در `data/prices_cache.json`
   ذخیره می‌کند. هندلرهای تلگرام فقط همین فایل را می‌خوانند، پس جواب‌دهی ربات به
   کاربر آنی است و دیگر هر درخواست باعث یک HTTP request جدید به tgju.org نمی‌شود.

2. **پنل مدیریت (`/admin`):** فقط برای آیدی‌های عددی داخل `ADMIN_IDS` در `.env`
   قابل مشاهده است. امکانات پنل:
   - لیست کاربرانی که با ربات چت خصوصی داشته‌اند
   - لیست گروه‌های تایید شده
   - لیست گروه‌های در انتظار تایید + دکمه‌ی تایید برای هر کدام
   - آمار کلی ربات (تعداد کاربر/گروه، آخرین به‌روزرسانی قیمت)

   وقتی ربات به یک گروه اضافه می‌شود، تا زمانی که ادمین از پنل تایید نکند، ربات
   در آن گروه به دستورات پاسخ قیمتی نمی‌دهد و فقط پیام «در انتظار تایید» می‌فرستد.

3. **دارایی‌های جدید:** پوند انگلیس (`/gbp`)، بیت‌کوین (`/btc`)، اتریوم (`/eth`) و
   تتر (`/usdt`) به دلار، یورو و طلای موجود اضافه شدند. برای افزودن دارایی جدید
   کافی است یک `Asset` دیگر به دیکشنری `ASSETS` در `config.py` اضافه کنی؛ بقیه‌ی
   کد (کیبوردها، دستورات، پنل و...) خودکار آن را پوشش می‌دهند.

4. **ذخیره‌سازی پایدار با Upstash Redis:** پلن رایگان Render دیسک *موقتی*
   (ephemeral) دارد؛ یعنی با هر ری‌استارت یا دیپلوی جدید، فایل‌های نوشته‌شده روی
   دیسک پاک می‌شوند. برای این‌که لیست کاربران/گروه‌ها و وضعیت تایید گروه‌ها از
   بین نرود، این‌ها را (نه کش قیمت را) در یک دیتابیس Redis رایگان روی Upstash
   نگه می‌داریم که مستقل از Render است.

   راه‌اندازی:
   1. یک اکانت رایگان در https://upstash.com بساز.
   2. یک دیتابیس Redis جدید بساز (پلن Free کافیست).
   3. از تب **REST API** همان دیتابیس، مقادیر `UPSTASH_REDIS_REST_URL` و
      `UPSTASH_REDIS_REST_TOKEN` را کپی کن و در `.env` بگذار.

   اگر این دو مقدار خالی بمانند، ربات crash نمی‌کند ولی به‌جای Redis از یک فایل
   محلی (`data/users.json` / `data/groups.json`) استفاده می‌کند که — دقیقاً مثل
   کش قیمت — روی Render رایگان با هر ری‌استارت پاک می‌شود. برای دیپلوی واقعی روی
   Render حتماً این دو متغیر را پر کن.

5. **`keep_alive.py` دست‌نخورده مانده** — همان فایلی است که قبلاً نوشته بودی و
   فقط یک وب‌سرور Flask سبک برای زنده نگه‌داشتن سرویس روی Render اجرا می‌کند.

## متغیرهای محیطی (`.env`)

```
BOT_TOKEN=...                     # توکن ربات از @BotFather
ADMIN_IDS=111111111,222222222     # آیدی عددی خودت (از @userinfobot بگیر)
UPSTASH_REDIS_REST_URL=...        # از پنل Upstash
UPSTASH_REDIS_REST_TOKEN=...      # از پنل Upstash
PROXY_URL=                        # اختیاری، فقط برای اجرای لوکال داخل ایران
```

## نصب و اجرا

```bash
pip install -r requirements.txt
python bot.py
```

## دستورات ربات

- `/start` — شروع و نمایش منو
- `/usd` `/eur` `/gbp` `/gold` `/btc` `/eth` `/usdt` — قیمت هر دارایی
- `/all` یا `/market` — نمایش همه‌ی قیمت‌ها
- `/admin` — پنل مدیریت (فقط برای آیدی‌های داخل `ADMIN_IDS`)

## نکته‌ی مهم درباره‌ی ری‌استارت‌های Render رایگان

پلن رایگان Render سرویس‌های وب را بعد از مدتی بی‌فعالیت می‌خواباند و گاهی هم
به‌صورت دوره‌ای ری‌استارت می‌کند تا منابع را آزاد کند. `keep_alive.py` فقط جلوی
خوابیدن به‌خاطر بی‌فعالیتی را می‌گیرد (به‌شرطی که چیزی — مثلاً یک سرویس رایگان
مثل UptimeRobot — هر چند دقیقه یک‌بار به آدرس سرویس پینگ بزند)؛ جلوی ری‌استارت‌های
دوره‌ای یا پاک شدن دیسک را نمی‌گیرد. به همین دلیل کش قیمت و (در صورت تنظیم‌نکردن
Upstash) لیست کاربران/گروه‌ها با هر ری‌استارت از نو ساخته می‌شوند.
